# 🚀 企業 AI 轉型行動清單：CoE 主管實戰手冊 [v1.2.0]
**指導準則**：以《企業生成式 AI 轉型全書》為核心基礎。

---

## 第零階段：評測先行與需求精準化 (第 1-2 週) [v1.1.0]
- [ ] **執行企業 AI 遺傳密碼診斷 (參照 Schema)**：利用五大屬性（風險、資產、邏輯、時間、人）對標二維座標定位。
- [ ] **建構黃金測試集 (參照 1.4)**：撰寫 50 題以上具備標準答案 (Ground Truth) 的業務考卷。
- [ ] **TAIDE 諮詢對標**：尋求在地專業模型的評測建議。
- [ ] **釐清目標與資料對標**：根據測試題反推所需資料清單。

## 第一階段：戰略對齊與共識建立 (第 3-6 週)
**關鍵任務：辨識戰位，防止盲目投入。**
- [ ] **高層訪談與戰略矩陣 (參照 1.2)**：區分「防禦型」與「進攻型」應用。
- [ ] **定義量化回報框架 (參照 1.3)**：設定三重 ROI 指標（效率、品質、機會）。
- [ ] **成立核心 CoE 小組 (參照 5.3)**：整合 IT、法務、數據與業務專家。

## 第二階段：知識底座的「基礎建設」 (第 5-8 週)
**關鍵任務：將數據轉化為 AI 的「企業大腦」。**
- [ ] **啟動資產化清滌 (參照 2.1)**：將核心文件 Markdown 化，建立知識數位孿生。
- [ ] **建構企業級 RAG 平台 (參照 2.2)**：建立具備準確上下文的檢索機制。
- [ ] **隱私堡壘與權限設計 (參照 2.3)**：設計脫敏與 Pre-retrieval 過濾。

## 第三階段：代理人代理與流程再造 (第 9-12 週)
**關鍵任務：實施首個高價值 Pilot 項目。**
- [ ] **代理人架構設計 (參照 3.1)**：設計具備「推理鏈」與「自我反思」的 Agent。
- [ ] **核心系統工具化 (參照 3.2)**：將 ERP/CRM API 轉換為 Agent 肢體。
- [ ] **流程大膽再造 (參照 5.1)**：實施「AI 做、人看 (HITL)」的新工作流。

## 第四階段：規模化、運維與文化重塑 (第 4 個月起)
**關鍵任務：建立自組織的數位勞動力工廠。**
- [ ] **啟動 AgentOps 監控 (參照 4.2)**：觀察思考路徑、成本與設置 Token 熔斷。
- [ ] **消除員工心理阻力 (參照 5.2)**：培訓員工轉型為「代理人管理者」。
- [ ] **跨代理人協作 (MAS) (參照 3.3)**：啟動多代理人數位部門協作。
- [ ] **法律與責任歸屬 (參照 4.3)**：擬定 AI 行為準則與責任鍊。

---

## 🔄 持續賦能機制：Wing Group 分享式循環 [v1.1.0]
- [ ] **成立 Wing Group**：由具備「五星特質」的積極分子組成敏感實驗小組。
- [ ] **實施產出切片與代號歸檔**：將分享內容轉為代號化的錄音摘要 (S1, S2...)，建立非同步知識流。
- [ ] **每週工具實驗 (Case Study)**：涵蓋生活（如旅行規劃）與公務，維持工具興奮感。
- [ ] **啟動「生活先行」補助計畫**：
    - 提供定額補助讓員工自選個人 AI 工具，對沖心理架構抗性。
    - 降低門檻，允許生活化應用以培養工具慣性。

## ✅ 流程有效性自我驗證 (Self-Validation) [v1.2.0]
*   **[ ] 職能位移**：先行者每日的「低價值重複勞動」是否位移給 AI？（先行者不累，流程才對）。
*   **[ ] 知識流速**：會議或雜記轉化為「可檢索資產」的時耗是否縮短？
*   **[ ] 去中心化**：非技術部門能否自主利用 AI 工具解決初步問題？
*   **[ ] 證據密度**：決策過程中調用歷史數據與交叉比對的頻率是否提升？
