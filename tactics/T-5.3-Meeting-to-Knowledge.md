# T-5.3-Meeting-to-Knowledge: 會議轉錄與 Wing Group 知識沉澱實務

## [Context] 問題場景
Wing Group (側翼小組) 或哈爸實驗室的運作核心在於「快速實驗、快速分享」。然而，會議中的大量高價值洞察（如河流探索經驗、跨域專家的隱性知識）往往隨風而逝，或停留在一長串難以檢索的對話中。傳統的人工記錄速度慢且容易遺漏細節。

## [Pattern] 解決模式：AI 驅動的「會議數位孿生」流程
透過錄音轉文字 (STT) 與 LLM 結構化摘要，達成「開完會即產出知識庫」的即時資產化。

1.  **實時錄音 (Capture)**：使用高品質錄音工具或線上會議錄製功能（如 Zoom/Google Meet）。
2.  **語音理解注入 (Voice Extraction)**：使用 NotebookLM 或 Gemini (Antigravity 讀取多維度資料) 同時上傳錄音檔與「議程背景/成員名單」。
3.  **結構化提煉 (Transformation)**：
    *   **核心亮點**：提取具備「里程碑」性質的決策與成果。
    *   **閃電 Demo 記錄**：詳細描述技術操作路徑（如 Jimmy 展示的 Antigravity 技巧）。
    *   **成員對位 (Member Spotlight)**：將發言者與其專業背景連結，建立社群人才庫。
4.  **人機協作校準 (Human-in-the-loop)**：人類管理員（如哈爸）檢查術語準確性、補齊截圖或現場照片。
5.  **歸檔與標籤化**：存入 `meetings/` 或 `posts/` 目錄，並添加 Hugo Frontmatter 方便檢索。

## [Artifacts] 實作資產：結構化摘要 Prompt 建議
```markdown
## 指令 (Prompt):
你是一位專業的 Wing Group 秘書。請分析上傳的會議錄音轉錄與議程背景。
1. 提取本次會議的 3 個核心里程碑。
2. 針對講者的技術 Demo (Jimmy/志全)，請詳細列出其核心技術點 (例如：MCP, Smart Code)。
3. 建立一個「成員介紹表格」，包含：發言者姓名、專業背景、發言關鍵觀點。
4. 整理出 3 個具體的「未來行動建議 (Next Steps)」。
5. 用繁體中文寫作，語氣要機智、條理分明且富有社群活力。
```

## [Failures] 坑洞紀錄
*   **角色誤判**：如果沒有提供「成員名單/背景」，AI 容易將發言者姓名聽錯或身分張冠李戴。**建議一定要先提供成員背景資料供 AI 對位**。
*   **語義斷鏈**：純音檔轉錄有時會丟失「視覺 Demo」的細節。**建議同步拍攝現場畫面或擷取 Demo 螢幕截圖**，並在結構化時告知 AI 圖片與內容的關聯。

## [Reference] 歸因案例
*   **哈爸實驗室雙周會#1**：結合 Jimmy (Antigravity) 與志全 (水利資料) 的分享，成功產出結構化的「翼組知識」。
